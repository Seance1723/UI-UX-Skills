# Landing Page Patterns & Chart Type Guide

Load this file when designing landing pages, marketing pages, or choosing data visualizations for dashboards.

---

## Part 1 — Landing Page Conversion Patterns

Each pattern describes a proven page structure. Match the pattern to the product type and conversion goal. Use the section order as a scaffold — apply brand tokens, copy, and components on top.

---

### Pattern 1 — Hero-Centric

**Conversion goal:** Create immediate desire and push to a single CTA.
**Best for:** Consumer products, SaaS, mobile apps, simple single-offer products.

**Structure:**
1. **Hero** — Bold headline (outcome, not feature). Subheadline. Primary CTA. Hero visual (product screenshot, lifestyle image, or animated demo).
2. **Social proof bar** — Logos or "X,000 users" (real numbers only).
3. **Core features** — 3 features, icon + title + 1-line description.
4. **Product showcase** — Screenshot or demo walkthrough.
5. **Testimonials** — 2–3 quotes with name, role, company.
6. **Pricing** — Clear, transparent, 2–3 tiers.
7. **Final CTA** — Repeat primary CTA with a secondary backup.

**Copy rule:** Hero headline must state an outcome ("Ship 10× faster"), not a category ("A development platform").

---

### Pattern 2 — Feature-Led

**Conversion goal:** Educate first, convert second. For considered purchases.
**Best for:** B2B SaaS, developer tools, complex products.

**Structure:**
1. **Hero** — Clear product category + differentiator. CTA to demo or trial.
2. **Logo bar** — Enterprise or recognizable customer logos.
3. **Feature deep-dives** — 4–6 features, each with a screenshot and 3-line description. Alternating left/right layout.
4. **How it works** — 3-step process diagram.
5. **Integration list** — "Works with your stack" logos.
6. **Social proof** — Case study card with company + metric.
7. **Pricing** — Feature comparison table.
8. **CTA** — Demo booking or free trial.

---

### Pattern 3 — Social Proof First

**Conversion goal:** Build trust before making any product claim.
**Best for:** New brands, high-skepticism categories (health, finance, legal).

**Structure:**
1. **Hero** — Modest headline. Social proof count ("Trusted by 12,000 professionals — real number").
2. **Customer logos** — Immediately below hero.
3. **Testimonial carousel or grid** — Above the fold-ish, prominent.
4. **Product features** — Now that trust is established.
5. **More testimonials / case studies** — With metrics.
6. **Security/compliance badges** (if relevant).
7. **CTA** — "Join them."

---

### Pattern 4 — Problem-Solution

**Conversion goal:** Identify the pain, then position the product as the only logical answer.
**Best for:** Products solving a specific painful problem. Especially effective for productivity, operations, and workflow tools.

**Structure:**
1. **Problem hook** — "Does this sound familiar?" or "Tired of [pain]?"
2. **Agitate** — List 3 symptoms of the problem. Make it feel recognized.
3. **Solution reveal** — "There's a better way." Introduce product.
4. **How it solves it** — Feature-to-problem mapping.
5. **Proof** — Testimonials from people who had the same problem.
6. **CTA** — "Stop [problem]. Start [solution]."

---

### Pattern 5 — Product Demo First

**Conversion goal:** Show, don't tell. Let the product speak.
**Best for:** AI tools, creative apps, anything with a compelling interactive demo.

**Structure:**
1. **Hero with live/embedded demo** — The product working in the hero itself.
2. **"Here's what you just saw" explainer** — Brief description of the demo.
3. **Key capabilities** — What else it can do.
4. **Use case tabs** — "For designers", "For developers", "For product managers."
5. **Testimonials from each persona**.
6. **Pricing or trial CTA**.

---

### Pattern 6 — Storytelling / Narrative

**Conversion goal:** Emotional connection before conversion.
**Best for:** Consumer brands, luxury, wellness, food, lifestyle, mission-driven companies.

**Structure:**
1. **Opening visual** — Lifestyle or brand-world hero. Minimal text.
2. **Brand story** — 3–4 paragraphs of genuine origin/mission.
3. **Product/offer introduced** — Natural transition from story.
4. **Product details** — Materials, process, sourcing (whatever is meaningful).
5. **Community/people** — Behind the brand or customers.
6. **CTA** — Invitation, not pressure.

---

### Pattern 7 — Comparison / Competitive

**Conversion goal:** Position against the alternative (usually the status quo or a named competitor).
**Best for:** Switching products, disruptors, "X but better" positioning.

**Structure:**
1. **Hero** — "Finally, a [category] that doesn't [competitor's problem]."
2. **Comparison table** — You vs. them. Be fair — only list real advantages.
3. **Deep feature sections** — Where you win.
4. **Testimonials from switchers** — "I switched from X because…"
5. **Migration offer** — Import, free migration, etc.
6. **CTA** — "Make the switch."

**Rule:** Never make false claims in comparison tables. Only list features you genuinely have. Users will verify.

---

### Pattern 8 — Waitlist / Coming Soon

**Conversion goal:** Capture interest before launch. Build anticipation.
**Best for:** Pre-launch products, exclusive launches, limited access.

**Structure:**
1. **Hero with countdown or "Coming [date]"** — Single focus.
2. **Teaser of what's coming** — Just enough to create desire.
3. **Email capture** — Primary action. Keep form to email only.
4. **Early access benefits** — What waitlist members get.
5. **Social proof** — "X people already on the waitlist" (if real).

---

### Pattern 9 — Pricing Page

**Conversion goal:** Help the user choose the right tier and commit.
**Best for:** SaaS, subscription products, membership products.

**Structure:**
1. **Headline** — "Simple, transparent pricing." No tricks promise.
2. **Toggle** — Monthly / Annual (annual should show savings).
3. **Pricing tiers** — 2–4 tiers. Highlight the recommended tier.
4. **Feature comparison table** — Below tiers for detail-oriented buyers.
5. **FAQ** — "What happens after my trial?" "Can I cancel anytime?"
6. **Enterprise CTA** — "Need more? Talk to us."
7. **Guarantee / Trust signal** — "Cancel anytime. No credit card required."

**Rules:**
- Never hide the price. If you can't show it, say "Custom pricing — contact sales" clearly.
- Always show what's included, not just what's different from tier to tier.
- Annual savings: show in % and in $ amount.

---

### Pattern 10 — Documentation / Developer Home

**Conversion goal:** Get developers into the docs and first integration fast.
**Best for:** APIs, SDKs, developer platforms.

**Structure:**
1. **Hero** — Product in one sentence. "Get started in 5 minutes." Code snippet.
2. **Quickstart** — Inline code block. Real runnable example.
3. **Key capabilities** — What you can build.
4. **Integration ecosystem** — "Works with X, Y, Z."
5. **Community / GitHub** — Stars, contributors, Discord.
6. **CTA** — "Read the docs." Never "Sign up" as the primary.

---

### Pattern 11 — Portfolio / Case Study

**Conversion goal:** Build credibility through demonstrated work.
**Best for:** Agencies, consultants, freelancers, service businesses.

**Structure:**
1. **Hero** — Name/studio + specialty. Best work preview.
2. **Work grid** — Projects with title, category, thumbnail.
3. **Case study detail** — Challenge → Process → Outcome (with metrics).
4. **About** — Brief, human, specific.
5. **Client logos** — Social proof.
6. **CTA** — "Start a project" or "Get in touch."

---

### Pattern 12 — E-commerce Category Page

**Conversion goal:** Help users discover and filter to the right product.
**Best for:** Retail, D2C, marketplace.

**Structure:**
1. **Category hero** — Seasonal or collection-led visual.
2. **Filter bar** — Sort, filter by attributes. Sticky on scroll.
3. **Product grid** — Cards with image, name, price, quick-add.
4. **Pagination or infinite scroll** — Clear affordance.
5. **Upsell section** — "Complete the look" or "Frequently bought together."

## Part 2 — Chart & Data Visualization Type Guide

Load this section when designing dashboards, analytics pages, or any data-heavy screen.

---

### Chart Selection Decision Tree

```
What is the user trying to understand?

COMPARISON
  ├── Between a few items (≤7)     → Bar Chart (vertical)
  ├── Between many items           → Horizontal Bar Chart
  ├── Over time                    → Line Chart
  └── Part of a whole              → Pie Chart (≤5 segments) / Donut Chart

TREND OVER TIME
  ├── Single metric                → Line Chart
  ├── Multiple metrics             → Multi-line Chart
  ├── Cumulative / stacked         → Stacked Area Chart
  └── Volume over time             → Bar Chart (time axis)

DISTRIBUTION
  ├── Spread of values             → Histogram
  ├── Statistical summary          → Box Plot
  └── Individual data points       → Scatter Plot

RELATIONSHIP / CORRELATION
  ├── Two variables                → Scatter Plot
  ├── Three variables              → Bubble Chart
  └── Categories vs. metric        → Heatmap

COMPOSITION (part of whole)
  ├── Static snapshot              → Pie / Donut (≤5 segments only)
  ├── Over time                    → Stacked Bar / Stacked Area
  └── Hierarchical                 → Treemap / Sunburst

FLOW / PROCESS
  ├── Sequential steps             → Funnel Chart
  ├── Flow between categories      → Sankey Diagram
  └── Network relationships        → Force-directed graph

GEOGRAPHIC
  └── Location-based data          → Choropleth Map / Dot Map

KPI / SINGLE METRIC
  └── Current value + context      → Metric Card with sparkline / Gauge
```

---

### Chart Type Specs

| Chart | Use when | Avoid when | Max data points |
|---|---|---|---|
| **Line** | Trends over time, continuous data | Comparing unrelated categories | 5–7 lines max |
| **Bar (vertical)** | Comparing categories, discrete data | More than 12 bars (use horizontal) | 12 bars |
| **Bar (horizontal)** | Long category names, ranked lists | Showing trends over time | 20 items |
| **Pie** | Part-of-whole, 3–5 categories only | More than 5 slices, comparing sizes | 5 segments |
| **Donut** | Same as pie + center metric | Same as pie | 5 segments |
| **Area** | Volume over time, cumulative data | Overlapping areas that obscure data | 3 series |
| **Stacked bar** | Part-of-whole over categories | Too many stack layers (>5) | 5 stack layers |
| **Scatter** | Correlation between two variables | Showing trends over time | 500 points |
| **Bubble** | 3-variable relationship | When bubble size differences are subtle | 50 bubbles |
| **Heatmap** | Density, frequency, correlation matrix | When exact values matter more than pattern | 100×100 cells |
| **Funnel** | Conversion pipeline, drop-off analysis | Anything not a sequential conversion | 7 stages |
| **Treemap** | Hierarchical part-of-whole | When exact sizes matter | 50 tiles |
| **Histogram** | Distribution of a single variable | Comparing multiple variables | 1 variable |
| **Metric card** | Single KPI + change + context | When the metric needs breakdown | 1 value |
| **Sparkline** | Trend at a glance, inline in table | When full axis detail is needed | Simple trend |
| **Gauge** | Current value vs. target/range | When absolute values matter more | 1 value |

---

### Dashboard Layout Rules

**Grid and density:**
- Cards: consistent padding (24px inner), consistent border-radius, consistent shadow.
- Spacing between cards: 16–24px.
- KPI metrics row: 4 cards across at desktop (2 at mobile).
- Charts: primary charts span 2/3 width; secondary charts span 1/3.

**Data typography:**
- Metric value: `--text-3xl` to `--text-4xl`, `font-weight: 700`, monospace optional.
- Metric label: `--text-sm`, `--color-text-secondary`.
- Chart labels: `--text-xs`, `--color-text-muted`.
- Table: `--text-sm`, `--text-base` for headings.

**Color in charts:**
- Use a consistent, accessible color sequence (not rainbow):
  1. Brand primary (--color-accent)
  2. Brand secondary (shifted hue)
  3. Neutral dark
  4. Semantic: success green, warning amber, error red
- Never use red/green as the only indicator in the same chart (colorblindness).
- Provide pattern + color differentiation for multi-series charts.

**Empty and loading states:**
- Skeleton: same size/shape as the chart it will replace.
- Empty: explain why it's empty + offer an action ("No data for this period — try adjusting the date range").
- Error: state what failed + offer retry ("Failed to load — Retry").

**Interaction patterns:**
- Tooltips on hover: show exact values, category label, date.
- Click-through on chart elements: drill down to detail view.
- Date range picker: always show current range prominently.
- Export: CSV and PNG download options for any chart.

---

### Anti-Patterns in Data Visualization

| Pattern | Why it's wrong | Fix |
|---|---|---|
| Truncated Y-axis (doesn't start at 0) | Exaggerates differences | Always start bar charts at 0 |
| Pie chart with >5 slices | Impossible to compare small slices | Use horizontal bar chart |
| 3D charts | Distorts visual perception of values | Never use 3D — always flat |
| Dual Y-axes with different scales | Implies false correlation | Use separate charts or normalize |
| Animations on every data update | Distracts, slows perception | Animate once on load only |
| Red/green only for success/failure | Colorblind users cannot distinguish | Add icon or pattern indicator |
| Showing raw numbers without context | Users can't judge if good or bad | Add target, average, or prior period comparison |
| Auto-refreshing without user control | Disorienting, loses user's place | Show "last updated" + manual refresh option |
